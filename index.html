P.L.A.N

Event Planning Mobile Application

" In order to streamline the event planning process, we are designing a mobile application that will help event planners organize their information in one location. "

  • Phase 1 - Sketches

  • Phase 2 - Medium Fidelity

  • Phase 3 - High Fidelity

Technology used for design:
Google Drive + Google Hangouts for meetings. Dropbox for sharing files. Adobe Creative Suite for creating mockups (Illustrator for designs, Photoshop for image altering).

Intended User Base:
Any event planners, professional or not

Defined Functionality (use cases):
A level, B level, C level --> TL;DR

Explorations:
Api - Google Maps, Foursquare, Eventbrite
Design Inspiration - Dribbble, Awwwards, Uiparade

Category Index + Category Show:
List of different categories of event planning that users can add. In index view, each category is like a widget that can be reorganized or removed. New categories can also be added. Each category can be clicked to view more details, leading to the category's show page. Show page includes all the details on a specific category.

Budget:
Show the total budget. Have a breakdown of money spent by categories. Have data visualization to allow users to see budget status at a glance.

Guest List + Guest Show:
Guest list page shows a list of all potential guests for an event. This page allows users to see a quick overview of people invited to the event, and rsvp management.

Add Collaborator:
Allow event planners to add people to a project with different access levels. Admins can make edits to an event, while others can only view.

Design Notes:
Play around with colors and layouts - test on 4 pages
No rounded corners
Flat Design

jéVoyage

Information Systems Junior Project

" Connect. Travel. Share. "

  • Final Design

  • Process

  • Logo Designs

  • User Testing

  • Branding

  • Recognition

3 Rounds of User Testing:
User testing was a very important part of our iterative development process. We asked a wide selection of peers including those who frequently travel abroad, who have studied abroad, and who have an interest in traveling abroad for their opinions on JeVoyage. We also asked potential users of JeVoyage to perform functional testing on early versions of the application.

Merchandise:
We wanted to present to our peers some ways in which JeVoyage could be marketed if it were to take off in the future. Includes t-shirts, water bottles, bookmarks, stickers (not pictured), and pens (not pictured).
